Unix Socket Programming

I implemented a library management network using C++. Five servers were created, "Client", "Main", "Science", "History" and "Literature". The Client and Main servers communicated over TCP for a reliable connection, while the Main server communicated with the three backend servers over UDP. Once the program begins, the Client is prompted to enter a username and password. The program parses through the input .txt file to determine if the username/password combination is in the system. If it is not, the user will be prompted to try again. Otherwise, the user gains access to the library system. If the admin username/password is entered, more access is given, as described later. Under the Client server, the user is next prompted to enter a book code for either the Science, History, or Literature department, which is sent to the Main server. The Main server checks the book code against the three backend servers, and sends status (avaliable or not avaliable) back to the Client. If the client is accessing the system as an Admin, the number of books left under that code will also appear. The number of books avaliable under the book code will decrease every time a client sends a request.

FPGA Programming

I became familiar with Modelsim and Xilinx Vivado while learning how to program the Nexys 7 FPGA. For one of my projects, I wrote VHDL code to implement a divider with cache. I wrote VHDL for the divider, cache, CAM and LRU stack, then used a testbench in Modelsim to verify functionality. I then synthesized the project and created a bitstream to send to the FPGA. Using UART, I sent a 4 bit hex number to the FPGA where the first 2 bits represented the dividend and the last 2 bits represented the divisor. I stepped through division using a button on the FPGA until the desired result appeared on the FPGA screen, where the first 2 bits represented the quotient and the last 2 bits represented the remainder.
